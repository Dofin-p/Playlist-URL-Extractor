# YouTube Playlist Extractor
デモ版 : https://dofin-p.github.io/Playlist-URL-Extractor/
## 概要

YouTubeの再生リストに含まれる全動画URLを、ワンクリックで一括取得・一覧表示するWebアプリケーションです。

---

## 開発の背景

NoteBookLMを用いた技術学習において、YouTubeの再生リストから動画URLを1件ずつ手動でコピーする作業が繰り返し発生しており、1回あたり数十分の工数が取られていました。

この非効率を解消するため、YouTube Data APIとGoogle Apps Script（GAS）を活用して本ツールを開発しました。結果として、これまで数十回のクリックを要していた作業が**約1分**に短縮され、学習そのものに時間を充てられるようになりました。

---

## システム構成

初期バージョン（V1）ではGAS単体で完結するスクリプトとして実装しましたが、UX向上と処理の見通しをよくすることを目的に、フロントエンド（HTML/Vanilla JS）とバックエンドAPI（GAS）を分離したSPA構成へ移行しました。

| レイヤー | 技術 |
|---|---|
| Frontend | HTML / CSS / Vanilla JavaScript |
| Backend (API) | Google Apps Script (GAS) |
| External API | YouTube Data API v3 |

---

## アーキテクチャ選定とセキュリティに関する考察

本ツールの構成上、フロントエンドのコード内にGASのエンドポイントURLが含まれており、外部からのリクエスト自体を完全に防ぐことはできません。この点については、以下の通りトレードオフを整理した上で現在のアーキテクチャを採用しています。

**APIキーの秘匿性について**  
YouTubeのAPIキーはGAS側にのみ保持しており、フロントエンドからYouTube APIを直接呼び出す構成はとっていません。GASをプロキシとして挟むことで、最も漏洩リスクを避けるべきYouTube APIキーの保護は担保しています。

**MVPとしての優先判断**  
本格的な商用サービスであれば、バックエンド側でのCORS制限やトークン認証を実装する必要があります。ただし本ツールは個人利用における業務効率化を目的としたため、開発速度を優先しました。万が一エンドポイントが悪用されAPIクオータが枯渇した場合は、GASのデプロイIDを再発行してエンドポイントURLを変更する運用でカバーする設計としています。
